.OBJETIVO DEL PROYECTO

     El objetivo del proyecto es dar una solución a los problemas de integración de personas con capacidades diferentes en el ámbito laboral, asistiéndolos para que puedan realizar el cobro de productos interactuando directamente con los clientes.

OBJETIVO DEL SISTEMA

        Dar soporte a la actividad de cobro de productos comercializados, contemplando apertura, cierre y actualización de caja.

LÍMITES

Desde: La apertura de una caja y la registración de un cobro.

Hasta: Cierre de la caja.

ALCANCES

  • Administrar usuarios.
  • Administrar dinero.
  • Administrar productos
  • Gestionar caja.
  • Gestionar cobro.
  • Emitir reportes de movimiento de caja (historial).

DESCRIPCIÓN DE CASOS DE USO

Nombre CU

REGISTRAR INICIO DE CAJA

Actores

Supervisor.

Objetivo

Registrar los billetes y monedas disponibles en la caja antes de realizar el inicio de la actividad.

Pre-Condición

Rol registrado.

Escenario
  1. El usuario selecciona su imagen para ingresar.
  2. El sistema solicita su password y el supervisor lo ingresa bien.

2.A El sistema solicita su password y el supervisor lo ingresa mal.

2.B El sistema solicita nuevamente el  password.

  1. El sistema muestra la opción de inicio de caja si el usuario es supervisor y no hay una caja ya abierta.
  2. Selecciona la opción Inicio de caja.
  3. Ingresa el detalle de cada tipo de dinero con su respectiva cantidad.
  4. El sistema registra el usuario, la cantidad y la fecha que se realizó el inicio de caja.
  5. Fin de caso de uso.

Post-Condición

Caja registrada con cada tipo de dinero disponible.

Observación

Nombre CU

REGISTRAR COBRO SEGÚN SELECCIÓN DE PRODUCTO

Actores

Operador.

Objetivo

Registrar el cobro permitiendo el ingreso de cada tipo de dinero recibido y visualizar cada tipo de billete y moneda entregados en concepto de vuelto.

Pre-Condición

Caja Abierta

Rol registrado como operador.

Escenario
  1. El usuario selecciona su imagen para ingresar.
  2. El sistema muestra la opción de inicio de actividad.
  3. El sistema muestra el listado de productos a seleccionar con una imagen acorde a la acción.
  4. El usuario selecciona el producto por unidad, la cantidad de veces deseadas.

              4.A El usuario selecciona el producto por peso.

              4.B El sistema solicita que ingrese el monto de dinero.

  1. El sistema visualiza el subtotal, con redondeo a 0,50 centavos sobre el total.
  2. El usuario selecciona el fin de la actividad selección de productos.
  3. El sistema visualiza el total a cobrar y solicita confirmación.
  4. El usuario confirma la finalización de la actividad de selección de productos.
  5. El sistema solicita que se ingrese la cantidad de cada tipo de dinero recibido.
  6. El usuario ingresa la cantidad de tipo de dinero recibido y confirma finalización de actividad.
  7. El sistema informa si el monto de dinero recibido es suficiente.
  8. El sistema informa el vuelto, visualizando tantas imágenes como billetes y monedas corresponda de cada tipo.

              9.A El sistema informa al operador la no existencia de vuelto.

              9.B El sistema llama al caso de uso “Registrar la actualización de caja”..

  1. El usuario confirma que se entregó el vuelto.
  2. El sistema registra el cobro actualizando la cantidad de cada tipo de dinero.
  3. Fin de caso de uso

Post-Condición

Cobro registrado y detalle de caja actualizado.

REGISTRAR CIERRE DE CAJA

Actores

Supervisor

Objetivo

Registrar el cierre de caja verificando la cantidad cada tipo de dinero disponible y visualizar el total correspondiente.

Pre-Condición

Caja Abierta.

Escenario
  1. El usuario selecciona su imagen para ingresar.
  2. El sistema solicita su password y el supervisor lo ingresa bien.

           2.A El sistema solicita su password y el supervisor lo ingresa mal.

           2.B El sistema solicita nuevamente el  password .

  1. El sistema muestra la opción de cierre de Caja.
  2. El Supervisor selecciona la opción Cierre de Caja
  3. El sistema visualiza la cantidad de cada tipo de dinero disponible en la caja y el total.
  4. El supervisor confirma cierre de actividad
  5. El sistema registra el cierre de caja con la información del usuario, las cantidades de tipo de dinero y la fecha que se realizó.
  6. Fin de caso de uso.

Caja cerrada.
